Sổ Tay Tra Cứu Lệnh Docker Toàn Tập

Tổng hợp chi tiết lệnh, mục đích và ví dụ thực tế.

1. Quản lý Images (Hình ảnh)

Images là bản thiết kế (blueprints) chỉ đọc, dùng để tạo ra Container.

Lệnh (Command) Chức năng & Ví dụ
docker pull <image> Tải image từ Docker Hub về máy.
Ví dụ: Tải nginx về máy.
docker build -t <name> . Xây dựng image từ file Dockerfile tại thư mục hiện tại.
Ví dụ: docker build -t myapp:v1 .
docker images Liệt kê danh sách các image đang có trong máy.
docker rmi <id/name> Xóa image khỏi ổ cứng để giải phóng dung lượng.

2. Quản lý Containers (Máy vận hành)

Container là phiên bản "sống" đang chạy của một Image.

⭐ Lệnh quan trọng nhất: Khởi tạo và Chạy
docker run -d --name web-app -p 8080:80 -v ./data:/data nginx
  • -d: Chạy ngầm (không chiếm terminal).
  • --name: Đặt tên cho container.
  • -p 8080:80: Mở cổng (Truy cập localhost:8080 sẽ vào cổng 80 của container).
  • -v: Gắn ổ cứng (Volume) để lưu dữ liệu lâu dài.
Lệnh (Command) Chức năng & Ví dụ
docker ps Xem danh sách container đang chạy.
docker ps -a Xem tất cả container (bao gồm cả đã tắt).
docker stop <id> Dừng container an toàn.
docker start <id> Bật lại container đã tắt.
docker rm <id> Xóa container (phải stop trước). Dùng -f để xóa cưỡng chế.
docker logs -f <id> Xem nhật ký (log) thời gian thực. Rất quan trọng để sửa lỗi (debug).
docker exec -it <id> bash Chui vào bên trong container để gõ lệnh (dùng sh nếu không có bash).
docker stats Xem mức sử dụng CPU/RAM của các container (giống Task Manager).

3. Quản lý Volume & Network

Lệnh (Command) Giải thích
docker volume create <tên> Tạo kho lưu trữ dữ liệu ảo, an toàn kể cả khi xóa container.
docker volume prune Xóa tất cả các volume không được sử dụng (Cẩn thận mất dữ liệu).
docker network create <tên> Tạo mạng nội bộ để các container kết nối với nhau bằng tên.
docker network ls Liệt kê các mạng hiện có.

4. Docker Compose (Quản lý dự án)

Dùng cho file docker-compose.yml để chạy nhiều container cùng lúc.

Lệnh (Command) Chức năng
docker compose up -d Dựng toàn bộ hệ thống (build, create network, run containers) ở chế độ chạy ngầm.
docker compose down Dừng và xóa toàn bộ container & network của dự án.
docker compose logs -f Xem log gộp của tất cả dịch vụ trong dự án.

5. Dọn dẹp hệ thống (System Prune)

⚠️ Cảnh báo: Các lệnh này xóa dữ liệu vĩnh viễn.
docker system prune Xóa container đã tắt, mạng không dùng và image "rác".
docker system prune -a Xóa sạch sành sanh mọi thứ (bao gồm cả image chưa chạy). Dùng để reset máy về trạng thái sạch.
docker system df Xem Docker đang chiếm bao nhiêu dung lượng ổ cứng.
